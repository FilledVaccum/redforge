CLI Reference

The redforge CLI provides four commands: scan, list-probes, report, and serve.

bash
redforge --help

  🔥 RedForge — LLM Red Teaming & Vulnerability Scanner

  Commands:
    scan         Run a vulnerability scan against a target LLM
    list-probes  List all available probes
    report       Manage and render saved scan reports
    serve        Start the REST API server

redforge scan

Run a vulnerability scan against a target LLM. Requires --authorization to acknowledge you have permission to test the target.

redforge scan [OPTIONS]
Run all or selected probes against a target LLM provider
--provider, -p TEXT
string
LLM provider. One of: openai, anthropic, gemini, bedrock, azure, mistral, ollama, rest
required
--model, -m TEXT
string
Model name/ID (e.g. gpt-4o, claude-opus-4-6). Uses provider default if omitted.
--authorization, -a TEXT
string
Authorization acknowledgment. owned | authorized | research
required
--probes TEXT
string
Comma-separated probe IDs. Default: all 47 probes. Example: RF-001,RF-007,RF-011
--system-prompt TEXT
string
System prompt for the target model
--format, -f TEXT
string
Output format: json | sarif | html | markdown
default: markdown
--output, -o PATH
path
Save report to file. If omitted, prints to stdout.
--config PATH
path
Load scan configuration from YAML or JSON file (promptfoo-style config runner)
--concurrency INTEGER
int
Concurrent probe executions
default: 3
--dry-run
flag
List probes that would run without executing them
--no-store
flag
Do not persist scan results to disk

Examples

bash
# Full scan of all 47 probes
redforge scan --provider openai --model gpt-4o --authorization research

# Critical probes only, save HTML report
redforge scan \
  --provider anthropic \
  --model claude-opus-4-6 \
  --authorization authorized \
  --probes RF-001,RF-007,RF-011,RF-038,RF-041,RF-042,RF-046 \
  --format html \
  --output report.html

# Local Ollama model
redforge scan \
  --provider ollama \
  --model llama3.2 \
  --authorization owned \
  --format json

# From YAML config file
redforge scan --config redforge.yaml

# Dry run to preview probes
redforge scan --provider openai --authorization research --dry-run

# Custom REST endpoint
export REST_API_URL="https://my-llm.example.com/v1/chat/completions"
export REST_API_KEY="my-key"
redforge scan --provider rest --authorization authorized

redforge list-probes

Display all available probes with their ID, severity, OWASP mapping, and description.

redforge list-probes [OPTIONS]
List all 47 probes with metadata
--owasp TEXT
string
Filter by OWASP LLM ID (e.g. LLM01)
--severity TEXT
string
Filter by severity: critical | high | medium | low
--tag TEXT
string
Filter by tag (e.g. injection, jailbreak)
--json
flag
Output as JSON for scripting
bash
# List all probes
redforge list-probes

# Filter by OWASP category
redforge list-probes --owasp LLM01

# Filter by severity
redforge list-probes --severity critical

# JSON output
redforge list-probes --json | jq '.[].id'

redforge report

redforge report SUBCOMMAND
Manage and render saved scan reports
list
List all saved scan reports
show PATH
Render and display a specific report file
convert PATH --format TEXT
Convert a JSON report to another format (html, sarif, markdown)
delete PATH
Delete a saved report
bash
# List all saved reports
redforge report list

# Show a report in terminal
redforge report show ./redforge-results/scan_20260325.json

# Convert to HTML
redforge report convert ./redforge-results/scan_20260325.json --format html

# Convert to SARIF for GitHub
redforge report convert ./redforge-results/scan_20260325.json --format sarif

redforge serve

Start the FastAPI REST API server for programmatic access and CI/CD integration.

redforge serve [OPTIONS]
Start the REST API server (FastAPI + uvicorn)
--host TEXT
string
Bind host
default: 127.0.0.1
--port INTEGER
int
Bind port
default: 8000
--reload
flag
Auto-reload on code changes (development mode)
--workers INTEGER
int
Number of uvicorn worker processes
default: 1
bash
# Start on default port 8000
redforge serve

# Public-facing with auth
export REDFORGE_API_KEY="secure-secret-key"
redforge serve --host 0.0.0.0 --port 9000

# Development with auto-reload
redforge serve --reload

# API docs available at:
# http://localhost:8000/docs      (Swagger UI)
# http://localhost:8000/redoc     (ReDoc)

CI/CD Integration

Use SARIF output to integrate with GitHub Code Scanning:

yaml
# .github/workflows/llm-security.yml
name: LLM Security Scan
on: [push, pull_request]

jobs:
  redforge:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install RedForge
        run: pip install "redforge[openai]"
      - name: Run LLM Security Scan
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: |
          redforge scan \
            --provider openai \
            --model gpt-4o \
            --authorization research \
            --format sarif \
            --output results.sarif
      - name: Upload SARIF to GitHub
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: results.sarif
