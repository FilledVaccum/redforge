OpenAI

bash
pip install "redforge[openai]"
export OPENAI_API_KEY="sk-..."

redforge scan --provider openai --model gpt-4o --authorization research
python
from redforge.adapters.openai_adapter import OpenAIAdapter

adapter = OpenAIAdapter(model="gpt-4o")
# Also supports: gpt-4-turbo, gpt-3.5-turbo, gpt-4o-mini
# Also supports OpenAI-compatible APIs via base_url param
adapter_compat = OpenAIAdapter(
    model="my-model",
    base_url="https://my-openai-compat.example.com/v1",
    api_key="my-key",
)

Ollama (Local)

Test locally-running models with zero API key requirements:

bash
# Start Ollama (separate terminal)
ollama serve
ollama pull llama3.2

# Scan local model
redforge scan --provider ollama --model llama3.2 --authorization owned

# Custom URL
export OLLAMA_BASE_URL="http://my-ollama-server:11434"
redforge scan --provider ollama --model mistral --authorization owned

Generic REST

Connect to any OpenAI-compatible REST endpoint:

bash
export REST_API_URL="https://my-llm.example.com/v1/chat/completions"
export REST_API_KEY="my-key"
export REST_MODEL="my-model-name"

redforge scan --provider rest --authorization authorized

Writing a Custom Adapter

python
from redforge.adapters.base import BaseAdapter, AdapterResponse
import time

class MyCustomAdapter(BaseAdapter):
    def __init__(self, my_client):
        self.client = my_client

    async def send(self, messages, **kwargs) -> AdapterResponse:
        start = time.monotonic()
        raw = await self.client.chat(messages)
        latency = int((time.monotonic() - start) * 1000)
        return AdapterResponse(
            content=raw.text,
            model="my-model",
            latency_ms=latency,
            tokens_used=raw.usage.total_tokens,
        )

    @classmethod
    def from_config(cls, config) -> "MyCustomAdapter":
        return cls(my_client=MyClient(config["api_key"]))
