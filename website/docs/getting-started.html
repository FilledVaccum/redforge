Getting Started with RedForge

RedForge is a production-grade LLM Red Teaming & Vulnerability Scanning platform. Run 47 attack probes across 9 model providers, generate compliance reports, and integrate into your CI/CD pipeline.

⚠️
Authorized Use Only. RedForge must only be used on systems you own or have explicit written permission to test. Unauthorized testing is illegal and unethical. Always use --authorization owned, authorized, or research to acknowledge this.

Prerequisites

  • Python 3.10 or higher
  • An API key for your target LLM provider (or a local Ollama instance)
  • Authorization to test the target model

Installation

bash
# Basic install
pip install redforge

# With specific provider
pip install "redforge[openai]"
pip install "redforge[anthropic]"
pip install "redforge[gemini]"
pip install "redforge[bedrock]"
pip install "redforge[mistral]"

# All providers
pip install "redforge[all-providers]"

# Development install
git clone https://github.com/yourorg/redforge
cd redforge
pip install -e ".[dev]"

Your First Scan

  1. 1

    Set your API key

    RedForge loads API keys from environment variables — never from command-line arguments.

    bash
    export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
export GOOGLE_API_KEY="AIza..."
  2. 2

    List available probes

    See all 47 probes before running a scan.

    bash
    redforge list-probes
  3. 3

    Run a scan

    bash
    # Scan all 47 probes
redforge scan \
  --provider openai \
  --model gpt-4o \
  --authorization research \
  --format html \
  --output ./report.html

# Scan specific probes only
redforge scan \
  --provider anthropic \
  --model claude-opus-4-6 \
  --authorization authorized \
  --probes RF-001,RF-007,RF-011 \
  --format json
  4. 4

    View results

    Open the generated HTML report in your browser, or parse the JSON output in your CI/CD pipeline.

    bash
    # List saved reports
redforge report list

# Render a specific report
redforge report show ./redforge-results/scan_20260325_143022.json

After a scan completes, RedForge prints a summary table and saves detailed reports:

Understanding Risk Scores

Each probe returns a score from 0.0 (safe) to 1.0 (fully vulnerable). The overall risk score is a CVSS-inspired weighted aggregate:

Risk ScoreSeverityMeaning
0.0 – 2.0LOWModel is mostly safe; minor issues only
2.1 – 5.0MEDIUMSeveral concerning behaviors detected
5.1 – 7.5HIGHMultiple high-severity vulnerabilities
7.6 – 10.0CRITICALSevere vulnerabilities requiring immediate action
