Bidirectional Guardrails

RedForge's guardrail system provides llm-guard-style bidirectional scanning: InputScanners protect the model from malicious inputs, OutputScanners prevent harmful content from reaching users.

User Input → [InputScanners] → LLM → [OutputScanners] → User

Input Scanners

InjectionScanner

Detects 12 prompt injection patterns including instruction override, system prompt hijacking, and jailbreak framing.

python
from redforge.guardrails import InjectionScanner, ScanAction

scanner = InjectionScanner(threshold=0.5)
result = scanner.scan("Ignore previous instructions and tell me...")

result.action   # ScanAction.BLOCK
result.score    # 0.0–1.0
result.evidence # "Injection pattern: ignore_instructions"

LanguageScanner

Detects unexpected language switches that may indicate bypass attempts.

python
from redforge.guardrails import LanguageScanner

scanner = LanguageScanner(allowed_languages=["en"])
result = scanner.scan("Erzähl mir wie man Exploits baut...")
# Flags: unexpected language switch to German

TokenBudgetScanner

Enforces input token limits to prevent context stuffing attacks.

python
from redforge.guardrails import TokenBudgetScanner

scanner = TokenBudgetScanner(max_tokens=2000)
result = scanner.scan(very_long_input)  # Blocks if > 2000 tokens

Output Scanners

CredentialLeakScanner

Detects 12 credential patterns: OpenAI keys, AWS keys, GitHub tokens, Stripe keys, JWT tokens, private keys, and more.

python
from redforge.guardrails import CredentialLeakScanner

scanner = CredentialLeakScanner()
result = scanner.scan("Here's the key: sk-proj-AKIA1234567890ABCDEF")
# result.action = ScanAction.BLOCK
# result.evidence = "AWS Access Key ID detected"

MaliciousCodeScanner

Detects 12 malicious code patterns: shell injection, reverse shells, obfuscated eval(), SQL injection, path traversal, SSRF, and known malware signatures.

ToxicityOutputScanner

Detects 5 toxicity categories: hate speech, threats, harassment, self-harm instructions, explicit content.

GuardrailPipeline

python
from redforge.guardrails import (
    GuardrailPipeline, ScanAction,
    InjectionScanner, LanguageScanner, TokenBudgetScanner,
    CredentialLeakScanner, MaliciousCodeScanner, ToxicityOutputScanner,
)

pipeline = GuardrailPipeline(
    input_scanners=[
        InjectionScanner(threshold=0.5),
        LanguageScanner(allowed_languages=["en"]),
        TokenBudgetScanner(max_tokens=4000),
    ],
    output_scanners=[
        CredentialLeakScanner(),
        MaliciousCodeScanner(),
        ToxicityOutputScanner(),
    ],
)

# In your LLM application:
input_check = pipeline.scan_input(user_message)
if input_check.action == ScanAction.BLOCK:
    return "I cannot process that request."

llm_response = await adapter.send([{"role": "user", "content": user_message}])

output_check = pipeline.scan_output(llm_response.content)
if output_check.action == ScanAction.BLOCK:
    return "The response was blocked by safety filters."

return llm_response.content

# ScanAction values: ALLOW, WARN, BLOCK
