REST API Reference

RedForge exposes a FastAPI-based REST API for programmatic scanning, CI/CD integration, and building custom security dashboards.

Overview

Base URL http://localhost:8000

Start the API server with:

bash
# Start server (default port 8000)
redforge serve

# Custom port and host
redforge serve --host 0.0.0.0 --port 9000

# With API key auth enabled
export REDFORGE_API_KEY="your-secret-key"
redforge serve

# Via Docker
docker run -p 8000:8000 -e REDFORGE_API_KEY=secret redforge/redforge:latest serve

Interactive API documentation is available at /docs (Swagger UI) and /redoc (ReDoc).

Authentication

When REDFORGE_API_KEY is set, all API requests must include a Bearer token:

bash
curl -H "Authorization: Bearer your-secret-key" \
     http://localhost:8000/v1/probes
ℹ️
If REDFORGE_API_KEY is not set, the API runs in unauthenticated mode. Never expose an unauthenticated RedForge API on a public network.

Rate Limiting

The API enforces rate limits to prevent abuse:

  • Scan endpoints: 10 requests/minute per IP
  • Read endpoints: 120 requests/minute per IP
  • 429 Too Many Requests is returned when exceeded with a Retry-After header

Error Codes

StatusCodeMeaning
400INVALID_REQUESTMissing required fields or invalid values
401UNAUTHORIZEDMissing or invalid API key
403FORBIDDENValid key but insufficient permissions
404NOT_FOUNDScan, probe, or report ID not found
422VALIDATION_ERRORRequest body fails Pydantic validation
429RATE_LIMITEDToo many requests
500INTERNAL_ERRORUnexpected server error

Scan Endpoints

POST /v1/scan Start a new vulnerability scan ⌄

Initiates a scan against a target LLM. Returns a scan ID immediately; use GET /v1/scan/{id} to poll for results.

Request Body

ParameterTypeRequiredDescription
providerstring✱ requiredLLM provider: openai, anthropic, gemini, bedrock, azure, mistral, ollama, rest
modelstring | nulloptionalModel name/ID. Uses provider default if omitted.
authorizationstring✱ requiredAuthorization type: owned | authorized | research
probe_idsarray[string]optionalSpecific probe IDs to run. Runs all 47 if omitted.
system_promptstring | nulloptionalSystem prompt for the target model
compliance_frameworksarray[string]optionalNIST_AI_RMF, EU_AI_ACT, ISO_42001
output_formatsarray[string]optionalDefault: ["json"]. Options: json, html, sarif, markdown
concurrencyintegeroptionalConcurrent probe executions. Default: 3. Max: 20.
max_payloads_per_probeinteger | nulloptionalLimit payloads per probe for faster scans

Example Request

bash
curl -X POST http://localhost:8000/v1/scan \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "provider": "openai",
    "model": "gpt-4o",
    "authorization": "research",
    "probe_ids": ["RF-001", "RF-007", "RF-011"],
    "compliance_frameworks": ["NIST_AI_RMF"],
    "output_formats": ["json", "html"],
    "concurrency": 5
  }'

Response

// 202 Accepted { "scan_id": "scan_20260325_143022_a1b2c3", "status": "running", "provider": "openai", "model": "gpt-4o", "probe_count": 3, "created_at": "2026-03-25T14:30:22Z", "poll_url": "/v1/scan/scan_20260325_143022_a1b2c3" }

Probe Endpoints

GET /v1/probes List all available probes ⌄

Query Parameters

ParameterTypeDescription
owasp_idstringFilter by OWASP ID (e.g. LLM01)
severitystringFilter: critical | high | medium | low
tagstringFilter by tag (e.g. injection)

Example Response

{ "probes": [ { "id": "RF-001-prompt-injection", "owasp_id": "LLM01", "mitre_atlas": "AML.T0051", "severity": "critical", "description": "Classic instruction override attacks...", "tags": ["injection", "owasp-llm01"], "payload_count": 20 } ], "total": 47 }
GET /v1/probes/{probe_id} Get probe details and sample payloads ⌄

Returns full probe metadata including description, OWASP/MITRE mappings, and sample (redacted) payloads.

Report Endpoints

GET /v1/reports/{scan_id} Get report metadata ⌄

Returns available report formats for a scan with download URLs.

GET /v1/reports/{scan_id}/{format} Download report in specific format ⌄

Path Parameters

ParameterTypeDescription
scan_idstringScan ID
formatstringjson | html | sarif | markdown

Returns the report file with appropriate Content-Type header. HTML reports have all raw model output sanitized.

scorefloatVulnerability score 0.0 (safe) to 1.0 (vulnerable)
